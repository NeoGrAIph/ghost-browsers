# Обзор интерфейса Camofleet UI

## Функциональные блоки и возможности

- **Левая панель управления.** Содержит статус воркеров с визуальной индикацией и форму запуска сессии, позволяющую выбирать воркера, headless/VNC режим, TTL, стартовый URL и произвольные метки.
- **Основной экран сессий.** Разделён на вкладки «Manage sessions» и «Session wall». На первой вкладке отображаются таблица активных сессий и инспектор выбранной сессии с управлением (touch/kill, копирование WebSocket, VNC в iframe). На второй — «video wall» с сеткой VNC превью.
- **Закреплённые превью.** Пользователь может закреплять VNC-сессии, формируя отдельную панель быстрого доступа.
- **Статистика.** В верхней панели отображаются суммарные метрики по статусам, обновляющиеся вместе со списком сессий.
- **Тема.** Есть переключатель светлой/тёмной темы с сохранением в `localStorage`.
- **Автообновление.** Списки воркеров и сессий периодически перезапрашиваются (polling 5с) и синхронно обновляют связанные блоки.

## Сильные стороны

- **Чёткая иерархия.** Сайдбар отвечает за контроль инфраструктуры и запуск, основная область — за мониторинг активных сессий.
- **Продуманная детализация.** Инспектор объединяет ключевые атрибуты сессии, быстрые действия и встроенный VNC.
- **Многозадачность.** Поддержка wallboard и закреплённых превью помогает одновременно отслеживать несколько сессий.
- **Согласованный визуальный язык.** Используются единые паттерны (панели, пилюли статусов, табы), что снижает когнитивную нагрузку.
- **Простое масштабирование.** Табличное представление и wallboard хорошо масштабируются при большом количестве сессий.

## Ограничения и слабые места

- **Статическое автообновление.** Используется REST polling, поэтому обновления приходят с задержкой и нет SSE/WebSocket.
- **Ограниченный контроль ресурсов.** Нет inline-редактирования прокси/лейблов, нет индикаторов загрузки для отдельных действий.
- **Невозможность гибко фильтровать.** Таблица не поддерживает комбинированные фильтры/поиск, что усложняет работу с большими пулами.
- **Мало контекста по ошибкам.** Ошибки отображаются одной строкой без рекомендаций по устранению.

## Идеи по улучшению и адаптации для Ghost Browsers

- **Гибрид SSE + polling.** Сохранить визуальные паттерны Camofleet, но подключить наше SSE-обновление для мгновенной синхронизации.
- **Расширенный фильтр.** Встроить текущие фильтры Ghost Browsers (по статусу, региону, прокси) в верхнюю панель рядом со статистикой.
- **Инспектор как центр управления.** Объединить детальную информацию, управление прокси и VNC в одной панели, добавить быстрые действия (копирование WebSocket, закрепление превью).
- **Закреплённые превью и wallboard.** Перенять механику pin/wallboard Camofleet, но использовать наши данные (snapshot, VNC URL) и локализованный интерфейс.
- **Последовательное брендирование.** Сохранить фирменные цвета Ghost Browsers в новых компонентах и поддержать существующий переключатель темы.
- **Улучшенный UX ошибок.** Добавить отдельный баннер для сетевых ошибок и состояния переподключения SSE.

Эти улучшения легли в основу переработанного интерфейса оператора Ghost Browsers: добавлены табличное представление, панель закреплённых сессий, стенд с живыми превью, а также обновлена панель деталей с действиями и копированием WebSocket.
