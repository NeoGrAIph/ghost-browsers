# Задание: пакет core

Общий пакет `core` задаёт контракт между Runner, Gateway, VNC Gateway и UI.
Перед реализацией изучите архитектурные и конфигурационные документы:
`docs/architecture.md`, `docs/configuration.md`, а также требования сервисов,
которые будут использовать модели (`services/gateway`, `services/runner`,
`services/vnc-gateway`, `apps/ui`).

## Минимальный объём работ
1. Согласовать схему сущностей `Runner`, `Session`, `SessionEvent` с ожиданиями
   всех сервисов. Фиксируем детальные поля, статусы, типы событий, а также
   инварианты (например, TTL токена ≤ 300 с, `updated_at ≥ created_at`).
2. Реализовать Pydantic-модели и экспортировать их через `core/__init__.py`.
   Каждая модель снабжается docstring’ами с примерами использования.
3. Реализовать модуль `websocket_bridge.py`: абстракцию моста для ретрансляции
   событий и in-memory реализацию для тестов и локальной разработки.
4. Покрыть модели и утилиты юнит-тестами (`tests/test_models.py`) на валидацию,
   сериализацию и корректную работу моста событий.
5. Обновить `AGENT_NOTES.md`: описать публичные интерфейсы моделей, зафиксировать
   принятые решения, ограничения и как запускать тесты.

## Качество и доработки
Старайтесь выполнять задачи так, чтобы не возвращаться к ним повторно. Если в
процессе дальнейшей разработки всплывут новые требования к `packages/core`,
дорабатывайте его аккуратно, сохраняя обратную совместимость.

**Требования к коду:** см. корневой `AGENTS.md` (docstring/JSDoc, инварианты,
тесты, обновление AGENT_NOTES).
